§ 09how this works

Everything above — the tokenizer, every matmul kernel, the continuous-batching scheduler, the paged KV cache, this HTTP server — is written in daslang, a statically-typed scripting language for games. No PyTorch, no llama.cpp, no CUDA: the LLM engine — kernels, IR generators, tokenizer, loaders — is ~26 000 lines of .das, JIT-compiled to native code through LLVM at startup, with its hot kernels auto-tuned per box on first run. The speech-recognition stack adds ~7 000, the Metal and Vulkan GPU backends ~15 000, and the server, scheduler, and this page ~4 000 more. The GPU tier offloads MoE expert stacks over Vulkan compute — hybrid by layer type, not layer count.

Source: GaijinEntertainment/daScript — the engine lives in modules/dasLLAMA, this server in utils/dasllama-server.
